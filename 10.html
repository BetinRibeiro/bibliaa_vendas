Vendas em Grupo: Orquestrando o Consenso

Em uma sala cheia, você não faz uma apresentação; você lidera uma discussão onde todos os nós (stakeholders) devem atingir o estado de COMMIT.

// Distributed Consensus (Raft/Paxos Style)
if (consensus_reached(decisor, influenciador, usuario, cético)) {
  return STATUS_200_OK;
} else {
  rebalance_cluster(); // Reajuste a energia da sala
}

Identifique os Papéis

Mapeie os endpoints: O Decisor (Root), o Influenciador (Proxy), o Usuário (Client) e o Cético (Firewall).

Regra do Contato Visual 2-2

Fale com uma pessoa por vez. Olhe nos olhos por 2-3 segundos (uma frase) e mude. Evite o loop de olhar apenas para o slide.

Tratamento de Exceções

Neutralize o cético antecipadamente. Traga a dúvida dele antes que ele a lance como um ataque ao sistema.

Os Mandamentos da Banca

Nunca foque em apenas uma pessoa (evite o Single Point of Failure).
Chegue cedo para mapear o terreno físico e os humores dos nós que chegam primeiro.
Não lute contra desafios; use o cluster: "Como vocês veem esse ponto que o colega trouxe?".
Use materiais físicos (Handouts) para reduzir a latência de atenção voltada apenas à tela.
Detecte e gerencie o "Seqüestrador de Reunião" (o nó que consome toda a largura de banda).

Scripts de Orquestração

Abertura de Sessão:

"Obrigado por estarem aqui. Sei que cada um tem uma visão diferente sobre o projeto, então meu objetivo é mostrar como nossa solução atende a cada uma dessas necessidades."

Health Check (Fechamento):

"Considerando o que discutimos, alguém aqui vê algum impedimento técnico ou financeiro para avançarmos agora?"
